# Обновление: озвучка, подсветка кликов, препроцессор речи

Если скилл стоял у тебя **без озвучки** — этот гайд добавляет её (плюс подсветку
кликов). Две команды:

```bash
git pull        # свежий код
npm run setup   # голосовой стек + база словоформ + синхронизация в ~/.claude
```

`npm run setup` идемпотентен — можно запускать сколько угодно раз. Он собирает
venv с голосовым движком (до ~2 ГБ моделей при первом разе), качает базу
словоформ и копирует свежий код в глобальную копию скилла
`~/.claude/skills/video-walkthrough/`. Дальше:

```bash
npm run video:voice   # проверка: в выводе должно быть «движок: silero»
```

## Что нового
- **Озвучка субтитров** — `npm run video:voice` (Silero, офлайн) или
  `npm run video:voice:hq` (голос Microsoft Svetlana). Работает пост-фактум по
  готовой паре `mp4`+`srt`, старые ролики озвучиваются без перезаписи видео.
- **Подсветка кликов** — в записи каждый клик отмечается кольцом в точке нажатия.
- **Препроцессор речи** — ударения, числа, словарь произношений терминов.
- **Скриншоты прохода** — `npm run screens` складывает чистые кадры до/после
  каждого шага в `screens/<сценарий>/` (банк для UX/UI-анализа). Заодно
  fast-режим больше не рисует плашку субтитра в DOM.
- **Английская озвучка** — `narrate.py` определяет язык по субтитрам каждого
  видео (`--lang auto|ru|en`): пишете шаги сценария по-английски — субтитры и
  голос будут английскими (silero `en_0` / edge `Aria` / say `Samantha`).
  Русский препроцессор (ударения, словарь, транслит) для английского
  пропускается. Эталон — `scenarios/example-en.spec.ts`. Документация теперь
  двуязычная: README.md — английский, README.ru.md — русский.
- **Фикс: mp4 для всех сценариев прогона.** Раньше финализация видео висела на
  `test.afterEach` уровня модуля — из-за кэширования модуля на воркер mp4
  получал только первый spec-файл, остальные молча оставались без видео (с
  одним сценарием не проявляется). Теперь финализация в teardown фикстуры
  `cap` — срабатывает для каждого теста. Обнови `qa/lib/fixtures.ts`.

## Скриншоты прохода: как включить в своём проекте

Съёмка живёт в общей обёртке `step()`, поэтому в проекте с каталогом `qa/`
нужно обновить библиотеку и добавить скрипт:

```bash
cp lib/caption.ts lib/fixtures.ts lib/steps.ts lib/screens.ts <проект>/qa/lib/
```

и в `<проект>/qa/package.json` в `scripts`:

```json
"screens": "SCREENS=1 playwright test --project=fast"
```

Дальше `npm run screens` — кадры появятся в `qa/screens/<сценарий>/`.

## Клик-эффект: почему его нет и как включить

Подсветка кликов **вшивается в кадр во время записи**, поэтому, в отличие от
озвучки, её нельзя добавить в готовое видео — ролик надо **перезаписать**:

```bash
npm run video
```

Если записываешь из своего проекта (каталог `qa/`), а не из репозитория скилла,
скопируй туда свежие файлы и перезапиши:

```bash
cp lib/click-marker.ts lib/fixtures.ts <проект>/qa/lib/
```

(`npm run setup` синхронизирует только глобальную копию `~/.claude`, до `qa/`
твоего проекта он не дотягивается — его путь ему неизвестен.)

## Если голос читает слово неправильно

Слова с мягким «е» вместо «э» (тег, плеер, менеджер), латиница, аббревиатуры —
это ожидаемо, лечится словарём `lib/accents.json`. Подробности — в разделе
«Озвучка» в [SKILL.md](.claude/skills/video-walkthrough/SKILL.md): там описан
рабочий цикл пополнения словаря и генератор форм `add-accent.py`.
