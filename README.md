# video-walkthrough-skill

**Скилл для [Claude Code](https://claude.com/claude-code): превращает один
Playwright-сценарий в автотест + видео-инструкцию с субтитрами по шагам.**

Тот же сценарий, что проверяет фичу, прогоняется в браузере со `slowMo` и
подписывает каждый шаг субтитром. Клики подсвечиваются кольцом в точке нажатия,
а субтитры можно **озвучить голосом**. На выходе — `.mp4` (+ `.srt`), который
служит одновременно **доказательством, что фича протестирована**, и
**видео-документацией** для заказчика/команды. Третий артефакт того же
сценария — **банк чистых скриншотов прохода** (`npm run screens`): пары кадров
до/после каждого шага для UX/UI-анализа.

> Один Gherkin-сценарий = спецификация + автотест + сценарий видео.

## Как это выглядит

```
step(cap, 'Вводим текст новой задачи', async () => {
  await page.getByTestId('task-input').fill('Записать видео-инструкцию')
})
```

Текст шага становится и заголовком в отчёте Playwright, и субтитром на видео.
Каждый субтитр держится на экране ≥5 c, чтобы человек успел прочитать.

## Быстрый старт (демо из коробки)

Требуется Node 18+, `ffmpeg` и `python3` в PATH.

```bash
npm install
npx playwright install chromium

npm test            # прогон без записи (проверка «зелёно/красно»)
npm run video       # запись → videos/example.mp4 (+ .srt)
npm run video:burn  # то же, но субтитры прожжены в кадр через ffmpeg

npm run video:voice     # озвучить субтитры (Silero, офлайн)
npm run video:voice:hq  # то же голосом Microsoft Svetlana (живее, но облако)

npm run screens     # чистые скриншоты до/после каждого шага → screens/<сценарий>/
```

Озвучка — отдельный шаг **поверх готовой пары `mp4`+`srt`**: разово поставь
голосовой стек (см. [Озвучка](#озвучка)), и старые ролики озвучиваются без
перезаписи видео.

Демо-приложение (`scenarios/demo/index.html`) — простой список задач; webServer
поднимает его сам. Готовый ролик — `videos/example.mp4`.

## Что внутри

```
.claude/skills/video-walkthrough/SKILL.md   # инструкция для Claude Code
lib/
  caption.ts       # субтитры-оверлей + сборка .srt (переживают навигацию)
  click-marker.ts  # подсветка кликов кольцом в точке нажатия (режим video)
  record.ts        # webm → mp4 (ffmpeg), сайдкар .srt
  fixtures.ts      # фикстура cap + финализация видео в режиме video
  steps.ts         # step() (субтитр+шаг+удержание), openApp(), dragSelect()
  screens.ts       # скриншоты прохода до/после шага (SCREENS=1, режим fast)
  narrate.py       # озвучка субтитров: Silero/edge/say, препроцессор речи
  accents.json     # словарь произношений терминов (Whisper→уиспер, тег→тэг)
  add-accent.py    # генератор форм слова для accents.json по базе словоформ
playwright.config.ts   # webServer + проекты fast|video
scenarios/
  example.feature      # человекочитаемый Gherkin
  example.spec.ts      # исполняемый сценарий (эталон для копирования)
  demo/index.html      # демо-приложение
```

## Подключение к своему проекту

1. Скопируй `lib/`, `playwright.config.ts`, `package.json` в каталог `qa/` своего
   репозитория; `.claude/skills/video-walkthrough/` — в корень репозитория.
2. `cd qa && npm install && npx playwright install chromium`.
3. В `playwright.config.ts` замени демо-`webServer` на запуск своего приложения
   **на отдельных портах с seed-данными** (шаблон — в комментарии внутри конфига).
4. Добавь в приложение ~5 `data-testid` на ключевые узлы сценария.
5. Пиши сценарии `qa/scenarios/US-XX-<slug>.{feature,spec.ts}` по образцу
   `example.*`. Имя файла = имя ролика.

Дальше Claude Code сам подхватит скилл по запросу вроде «запиши видео-инструкцию
по фиче US-42».

## Подсветка кликов

В режиме `video` каждый клик подсвечивается расходящимся кольцом с точкой в месте
нажатия (`lib/click-marker.ts`) — зритель ясно видит, куда нажал сценарий. Маркер
инжектируется через `addInitScript`, поэтому переживает навигацию и появляется
даже если приложение гасит событие (слушатель в capture-фазе). Вшивается прямо в
кадр, как субтитры; в быстрых прогонах `npm test` не участвует.

## Скриншоты прохода

`npm run screens` прогоняет те же сценарии в fast-режиме и складывает в
`screens/<сценарий>/` пары кадров `NN-before-…png` / `NN-after-…png` — состояние
интерфейса до и после каждого шага, имена из текстов шагов. Кадры **чистые**:
без плашки субтитра и маркера клика (в fast-режиме оверлеи не рисуются). Это
банк скриншотов для UX/UI-анализа продукта; прогон перезаписывает каталог
сценария целиком, историю ведёт git.

## Озвучка

`npm run video:voice` синтезирует речь по `.srt` и подмешивает её в `mp4` по
таймкодам субтитров (ffmpeg `adelay`). Работает **пост-фактум по паре mp4+srt** —
аудиодорожка заменяется целиком, повторный прогон идемпотентен. Это выполнение
кода, токены не тратятся.

**Движки** (`--engine auto|silero|edge|say`):

| Движок | Голос | Плюсы | Минусы |
|---|---|---|---|
| `silero` (по умолчанию) | `xenia` | офлайн, детерминированно, бесплатно | интонация ровнее |
| `edge` (`video:voice:hq`) | `ru-RU-SvetlanaNeural` | живая интонация из коробки | облако, неофициальный API |
| `say` | macOS `Milena` | без установки | звучит роботом |

**Читаемость речи.** Субтитры остаются чистыми; перед синтезом текст проходит
препроцессор:

- **Ударения** — RUAccent (словарь + омографы по контексту).
- **Числа и единицы** — «± 0.5 с» → «плюс-минус ноль целых пять десятых секунды».
- **Термины** — словарь `lib/accents.json` кириллицей (`Whisper` → «уиспер»,
  `тег` → «тэг», `менеджер` → «мэнэджэр»). Формы слова генерируются
  `add-accent.py` по базе словоформ, а не руками.
- **Незнакомая латиница** — страхуется транслитерацией (runorm), чтобы слово не
  выпало из озвучки молча.

**Выразительность (silero).** Паузы по пунктуации вставляются автоматически
(SSML). Смысловые акценты — sidecar `<видео>.speech.json` рядом с srt: указанное
ключевое слово читается медленнее с микропаузами. Заполняется при авторинге
сценария.

**Установка голосового стека** — одной командой (идемпотентно):

```bash
npm run setup
```

Собирает venv, качает базу словоформ и синхронизирует код в глобальную копию
скилла. Модели скачиваются в кэш при первом синтезе. Без стека `narrate.py`
откатывается на системный `say -v Milena`.

> Уже пользуешься скиллом, но без озвучки? `git pull && npm run setup` —
> подробности в [UPGRADING.md](UPGRADING.md).

## Принципы

- **Видео — побочный продукт теста.** Сценарий пишется в любом случае; запись —
  это флаг `--project=video`.
- **Проверка только по ассертам** (`expect`). Записанное видео мультимодально
  никогда не разбирается — это дорого и не нужно. Видео для человека, ассерты для
  машины.
- **Стабильные `data-testid`**, изолированный **seed**, тест-стенд на **отдельных
  портах** — детерминизм и никакого влияния на рабочую среду.

## Настройки

| Переменная | Смысл | По умолчанию |
|---|---|---|
| `STEP_HOLD_MS` | Минимум времени субтитра на экране (мс) | `5000` |
| `BURN_SUBS=1` | Прожечь `.srt` в кадр через ffmpeg | выкл (оверлей) |
| `slowMo` (в конфиге) | Пауза между действиями для видео | `300` мс |
| `--engine` (у `narrate.py`) | Движок озвучки: `auto`/`silero`/`edge`/`say` | `auto` |
