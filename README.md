# video-walkthrough-skill

**Скилл для [Claude Code](https://claude.com/claude-code): превращает один
Playwright-сценарий в автотест + видео-инструкцию с субтитрами по шагам.**

Тот же сценарий, что проверяет фичу, прогоняется в браузере со `slowMo` и
подписывает каждый шаг субтитром. На выходе — `.mp4` (+ `.srt`), который служит
одновременно **доказательством, что фича протестирована**, и **видео-документацией**
для заказчика/команды.

> Один Gherkin-сценарий = спецификация + автотест + сценарий видео.

## Как это выглядит

```
step(cap, 'Вводим текст новой задачи', async () => {
  await page.getByTestId('task-input').fill('Записать видео-инструкцию')
})
```

Текст шага становится и заголовком в отчёте Playwright, и субтитром на видео.
Каждый субтитр держится на экране ≥5 c, чтобы человек успел прочитать.

## Быстрый старт (демо из коробки)

Требуется Node 18+, `ffmpeg` и `python3` в PATH.

```bash
npm install
npx playwright install chromium

npm test          # прогон без записи (проверка «зелёно/красно»)
npm run video     # запись → videos/example.mp4 (+ .srt)
npm run video:burn  # то же, но субтитры прожжены в кадр через ffmpeg
```

Демо-приложение (`scenarios/demo/index.html`) — простой список задач; webServer
поднимает его сам. Готовый ролик — `videos/example.mp4`.

## Что внутри

```
.claude/skills/video-walkthrough/SKILL.md   # инструкция для Claude Code
lib/
  caption.ts     # субтитры-оверлей + сборка .srt (переживают навигацию)
  record.ts      # webm → mp4 (ffmpeg), сайдкар .srt
  fixtures.ts    # фикстура cap + финализация видео в режиме video
  steps.ts       # step() (субтитр+шаг+удержание), openApp(), dragSelect()
playwright.config.ts   # webServer + проекты fast|video
scenarios/
  example.feature      # человекочитаемый Gherkin
  example.spec.ts      # исполняемый сценарий (эталон для копирования)
  demo/index.html      # демо-приложение
```

## Подключение к своему проекту

1. Скопируй `lib/`, `playwright.config.ts`, `package.json` в каталог `qa/` своего
   репозитория; `.claude/skills/video-walkthrough/` — в корень репозитория.
2. `cd qa && npm install && npx playwright install chromium`.
3. В `playwright.config.ts` замени демо-`webServer` на запуск своего приложения
   **на отдельных портах с seed-данными** (шаблон — в комментарии внутри конфига).
4. Добавь в приложение ~5 `data-testid` на ключевые узлы сценария.
5. Пиши сценарии `qa/scenarios/US-XX-<slug>.{feature,spec.ts}` по образцу
   `example.*`. Имя файла = имя ролика.

Дальше Claude Code сам подхватит скилл по запросу вроде «запиши видео-инструкцию
по фиче US-42».

## Принципы

- **Видео — побочный продукт теста.** Сценарий пишется в любом случае; запись —
  это флаг `--project=video`.
- **Проверка только по ассертам** (`expect`). Записанное видео мультимодально
  никогда не разбирается — это дорого и не нужно. Видео для человека, ассерты для
  машины.
- **Стабильные `data-testid`**, изолированный **seed**, тест-стенд на **отдельных
  портах** — детерминизм и никакого влияния на рабочую среду.

## Настройки

| Переменная | Смысл | По умолчанию |
|---|---|---|
| `STEP_HOLD_MS` | Минимум времени субтитра на экране (мс) | `5000` |
| `BURN_SUBS=1` | Прожечь `.srt` в кадр через ffmpeg | выкл (оверлей) |
| `slowMo` (в конфиге) | Пауза между действиями для видео | `300` мс |
